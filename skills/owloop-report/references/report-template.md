# Independent HTML Report Template for Owloop

This is a starting template for generating a self-contained HTML report without relying on `lavish` or any external report generator.

## Full HTML Template

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Owloop Run Report — {{date}}</title>
  <style>
    :root {

      --border: #e0e0e0;
    }
    body {
      font-family: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;

    }

    h2 { margin-top: 2rem; color: #333; }

    </tbody>
  </table>

  <ul>

  <ol class="next-steps">

    {{/next_steps}}
  </ol>

  <hr>
  <p style="color: var(--gray); font-size: 0.9rem;">
    Generated by owloop-report skill · Independent of lavish.
  </p>

```

## How to Use

1. Read the latest owloop log from `.owloop/logs/`.
2. Extract the data: specs, statuses, blockers, verification commands.
3. Replace the `{{placeholder}}` values in the template.
4. Write the result to `.owloop/reports/owloop_report.html` or a user-specified path.
5. If the environment supports it, offer to open the file in a browser.

## Design Notes

- The template uses only inline CSS — no external dependencies.
- It uses CSS variables for the owloop amber brand color.
- Status classes are `.status-done`, `.status-blocked`, `.status-failed`.
- Keep the report under one page of scrolling if possible; use summary cards and tables.
